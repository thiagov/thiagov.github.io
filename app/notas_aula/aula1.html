Exemplos - Aula 1

Primeiro programa em C

Nosso primeiro programa em C é bastante simples: ele somente escreve na tela a mensagem "Ola mundo!". Segue a implementação desse programa:

#include <stdio.h> #include <stdlib.h> int main(int argc, char *argv[]) { printf("Ola mundo!"); return 0; }

Se você compilar e executar o programa acima, obterá a seguinte saída: 

    Ola mundo!

O que significa cada linha desse programa?

Linhas 1-2
Essas duas linhas começam da mesma forma: #include<...>. Vamos falar mais sobre esse tipo de comando no decorrer do curso. Inicialmente, no entanto, você deve saber que essas linhas servem para adicionar funcionalidades em nossos programas C. Para utilizar a função de imprimir na tela, por exemplo (printf), precisamos da biblioteca stdio.h. As linhas 1 e 2 estão incluindo duas bibliotecas em nosso programa: stdio.h e stdlib.h. A primeira nos dá acesso à função printf(), e a segunda nos da acesso à função system("PAUSE").
Linha 4
Nessa linha declaramos uma função, que chamamos de main(). A função main() é a função principal de qualquer programa em C. A execução de um programa em C sempre começa a partir do main(), e todo programa em C deve possuir essa função. O código da função é tudo aquilo que fica entre as chaves: {}. Um código fora das chaves não será executado. Veremos mais tarde como trabalhar com funções detalhadamente.
Linhas 5
Print, em inglês, significa imprimir. É exatamente o que o comando da linha 5 faz. Ele imprime uma mensagem na tela. O comando printf() é uma função que imprime a mensagem colocada entre parênteses na tela. A mensagem colocada entre parênteses deve estar entre aspas duplas (""). Veremos mais tarde que um texto entre aspas duplas é chamado, em programação, de String. Nesse caso, estamos imprimindo a String "Ola mundo!" na tela.
Linhas 6
Essa linha é utilizada para finalizar a função main(). Entraremos em detalhes sobre esse comando mais tarde. Por enquanto, devemos saber que nossa função main() sempre terminar com um return 0. Vale notar que comandos escritos abaixo do return 0 e antes do fechamento das chaves não são executados pelo nosso programa!
